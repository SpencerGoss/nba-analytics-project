The module must implement:

```python
REFEREE_DATA_DIR = "data/raw/external/referee_crew/"
OUTPUT_PATH = "data/features/referee_features.csv"

def build_referee_features(
    referee_data_dir: str = REFEREE_DATA_DIR,
    team_game_log_path: str = "data/processed/team_game_logs.csv",
    output_path: str = OUTPUT_PATH,
) -> pd.DataFrame:
    """
    Build per-game referee foul-rate features from scraped crew assignments.

    For each game with referee data:
    1. Look up each referee's historical FTA/game rolling average
    2. Average across the 3-ref crew to get crew-level foul rate
    3. Compute pace impact: crew's historical average possessions vs league average

    Returns DataFrame with columns:
        game_date, game_id_bref, home_team, away_team,
        ref_crew_fta_rate_roll10, ref_crew_fta_rate_roll20,
        ref_crew_pace_impact_roll10

    TRAINING PATH ONLY for historical games.
    NaN for games without referee data (SimpleImputer handles it).
    """
```

Implementation details:
- Load all CSV files from referee_data_dir (glob pattern: `referee_crew_*.csv`)
- Combine into a single DataFrame of all scraped referee assignments
- For each referee, compute a rolling average FTA/game from team_game_logs (using the games they officiated):
  - Join referee assignments to team game logs on (game_date, team abbreviation) to get FTA for each game
  - Each game produces TWO FTA values (one per team), average them for that game's ref FTA
  - For each referee, compute rolling 10-game and 20-game mean of per-game FTA (shift(1) to prevent leakage)
- For each game's crew (3 refs), average their individual rolling FTA rates to get crew-level rate
- Pace impact: compare crew's rolling average total possessions (from team game logs) vs league-wide average
- Output columns use NaN (not zero) for pre-scrape games per Pitfall 6 in RESEARCH.md — zero would inject false signal
- Save to data/features/referee_features.csv

Key design decisions:
- Separate module (not in team_game_features.py) per RESEARCH.md Open Question 4 recommendation
- Rolling window uses shift(1) to prevent leakage (NFR-1 compliance)
- Default to NaN when no referee data exists (model's SimpleImputer will mean-impute)
