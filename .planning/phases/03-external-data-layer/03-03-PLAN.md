```python
"""
NBA Pre-Game Injury Report Fetcher
====================================
Fetches the official NBA pre-game injury report for live inference.

INFERENCE PATH ONLY — never import or call this module from
build_injury_proxy_features() or any training code path.
The training path uses historical game-log-based proxy from
src/features/injury_proxy.py. These two code paths must NEVER share inputs
(FR-4.4).

Usage:
    from src.data.external.injury_report import get_todays_nba_injury_report
    df = get_todays_nba_injury_report()
"""

OUTPUT_DIR = "data/raw/external/injury_reports/"

def get_todays_nba_injury_report(
    output_dir: str = OUTPUT_DIR,
    save_snapshot: bool = True,
) -> pd.DataFrame:
    """
    Fetch current NBA official pre-game injury report.

    INFERENCE PATH ONLY. Never use for training feature construction.

    Strategy:
    1. Try nba_api LeagueInjuryReport endpoint (primary — no URL guessing)
    2. If API fails, try PDF fallback with common time slots
    3. Return structured DataFrame or empty DataFrame on total failure

    Returns:
        DataFrame with columns: player_name, team, player_status, reason, game_date
        Filtered to Out, Questionable, Probable statuses.
    """
```

Primary source (nba_api endpoint):
- Use `nba_api.stats.endpoints.leagueinjuryreport.LeagueInjuryReport` with `league_id="00"`, `season_type="Regular Season"`
- Sleep 1 second before API call (polite to NBA API)
- Normalize column names to lowercase
- Filter to `player_status in ["Out", "Questionable", "Probable"]`
- This is essentially the same logic as the existing `get_todays_injury_report()` in injury_proxy.py, but as a standalone module following the `src/data/get_*.py` pattern

PDF fallback (when API endpoint fails):
- Try common time slots in reverse order: ["06_00PM", "05_00PM", "01_30PM", "11_00AM", "06_00AM"]
- Use today's date formatted as YYYY-MM-DD
- Download PDF via requests.get() into io.BytesIO (do NOT save PDF to disk — per RESEARCH.md anti-pattern)
- Parse with pdfplumber: `pdf.pages[i].extract_table()`, skip header row per page
- Expected columns: game_date, game_time, matchup, team, player_name, current_status, reason
- Normalize column names to lowercase
- Filter to relevant statuses
- If all time slots return 404/403, return empty DataFrame with warning

Snapshot saving:
- If save_snapshot=True, save the fetched DataFrame to `data/raw/external/injury_reports/injury_report_{YYYYMMDD}_{HHMM}.csv`
- This builds a historical archive of injury reports for future analysis
