  <action>
Create `src/models/ats_backtest.py` mirroring the structure of `src/models/backtesting.py`:

1. **Config:**
   - `MIN_BACKTEST_GAMES = 500` (hard minimum -- raise ValueError if fewer)
   - `ATS_FEATURES_PATH = "data/features/game_ats_features.csv"`
   - `REPORTS_DIR = "reports"`

2. **`compute_roi_flat_110(covers: pd.Series) -> dict`:**
   - Standard -110 spread bet ROI calculation
   - Excludes NaN (pushes) via `dropna()`
   - Hard guard: `if len(valid) < MIN_BACKTEST_GAMES: raise ValueError(f"Backtest requires >= {MIN_BACKTEST_GAMES} games, got {len(valid)}.")`
   - Wins pay +$100 on $110 stake, losses cost $110
   - Return dict: n_bets, wins, losses, hit_rate, net_units, roi
   - Use float()/int() casts for JSON serializability (NFR-3)

3. **`compute_clv_spread(bet_spread, closing_spread, bet_side) -> float`:**
   - CLV = difference between our spread and closing spread from bettor's perspective
   - For home bets: CLV = bet_spread - closing_spread (positive if we got more points)
   - For away bets: CLV = closing_spread - bet_spread
   - NOTE: Kaggle dataset may only have one spread column (likely opening). If no separate closing spread is available, document this limitation and compute CLV as 0.0 (opening-to-opening is always 0). Add a `closing_spread` parameter that defaults to `opening_spread` when no closing data exists.

4. **`run_ats_backtest(ats_features_path=ATS_FEATURES_PATH, value_bet_only=False, threshold=0.05) -> dict`:**
   - Load `game_ats_features.csv`
   - Drop NaN covers_spread (pushes)
   - Load ATS model and make predictions on all games
   - Compute per-game: covers_spread_pred, covers_spread_actual, spread, edge (model prob vs implied)
   - **Two backtest modes:**
     a. `value_bet_only=False`: Backtest ALL games the model predicts (baseline ATS hit rate)
     b. `value_bet_only=True`: Backtest ONLY games flagged as value bets (edge > threshold)
   - For each mode, compute:
     - Overall hit rate (% of ATS predictions correct)
     - ROI via `compute_roi_flat_110()`
     - Average CLV
     - Per-season breakdown: season, n_games, hit_rate, roi, avg_clv
   - 500-game guard: raise if total backtest games < 500
   - Return dict with summary + per_season DataFrame

5. **`write_backtest_reports(results: dict, reports_dir=REPORTS_DIR)`:**
   - Create `reports/` directory if needed
   - Write `reports/ats_backtest.csv` -- per-season breakdown (season, n_games, wins, losses, hit_rate, roi, avg_clv)
   - Write `reports/ats_backtest_summary.txt` -- human-readable summary:
     - Total games backtested
     - Overall hit rate, ROI, average CLV
     - Per-season table
     - Value-bet filtered results (if available)
     - Model metadata (type, n_features, training date)
   - ASCII only in output (Pitfall 7)

6. **`run_ats_backtest()` main flow:**
   - Run baseline backtest (all games)
   - Run value-bet filtered backtest (edge > threshold)
   - Write both sets of results to reports
   - Print summary to console

7. Make runnable: `if __name__ == "__main__"` calls `run_ats_backtest()` and `write_backtest_reports()`.
