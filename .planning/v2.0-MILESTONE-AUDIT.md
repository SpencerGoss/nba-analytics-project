## Critical Gaps (Blockers for milestone completion)

### 1. DATA-01 / FEAT-01: Kaggle Injury Dataset Never Ingested

The original v2.0 plan called for replacing the rolling-average injury proxy with real Kaggle injury records (2016-2025). This was the primary injury improvement goal for v2.0.

- Phase 6 built a daily PDF fetcher for the inference path -- this is useful but does not satisfy DATA-01
- No Kaggle download script was ever written
- FEAT-01 depends on DATA-01 -- real injury absence features cannot be built without the dataset
- The training pipeline still uses injury_proxy.py (rolling game log averages)

### 2. MODEL-03: Ensemble/Stacking Layer Not Implemented

Phase 9 planned an ensemble/stacking layer (`get_strong_value_bets()`) combining game outcome and ATS signals for value-bet detection.

- value_bet_detector.py exists but uses single-model threshold comparison
- No stacking classifier (sklearn StackingClassifier or equivalent) in any model file
- get_strong_value_bets() function does not exist anywhere in the codebase
- Phase 9 summary acknowledged this was deferred -- it was never completed

### 3. MODEL-01: 68% Target Unconfirmed

The 68% game outcome accuracy target is claimed in commit 3764abf but is not confirmed by any report artifact.

- Holdout backtest shows 64.8% (2023-24) and 66.7% (2024-25)
- game_outcome_metadata.json is stale (pre-lineup run)
- game_outcome_model_calibrated.pkl was updated but metadata.json was not regenerated
- 68% may have been measured on combined holdout vs per-season; requires rerunning evaluation
